# 电信 688 融合套餐配套方案说明（AI算力末端交付 / VS Code 能力交付）

## 简要说明（通俗版）

这是一项给“电信 688 融合套餐”配套的落地服务：把 AI 算力真正送到客户电脑上，让客户在公司网络环境里打开 VS Code 就能稳定使用（能装、能连、能同步）。

我们具体做三件事：

- **装好 VS Code**：完成软件安装与基础配置，并处理常见网络阻碍。
- **开通 GitHub**：协助注册/登录 GitHub 账号，确保账号可用。
- **装好电信定制 V Kard 插件**：它相当于一个 Proxy（代理通道），让 VS Code 在受限网络下也能正常访问并保证 Git 同步准确有效。

## 1. 背景与定位

- **电信侧产品**：688 融合方案（电信企业套餐）。
- **我方配套产品**：AI 算力的“末端交付”，以 **VS Code（VSC）工作台能力交付** 为载体，帮助客户在真实网络环境下“能装、能连、能用、能同步”。

简而言之：电信提供“融合套餐 + 连接能力”，我方提供“AI算力落地到终端的可用性”。

## 2. 交付目标

- 在客户终端完成 VS Code 可用环境搭建。
- 打通 GitHub 使用路径，确保账号可用、Git 同步稳定。
- 通过电信定制 **V Kard Extension**（Proxy 能力）规避网络阻碍，使 VS Code 在企业网络中可持续稳定使用。

## 3. 交付范围（VSC 能力交付定义）

### 3.1 VS Code 软件安装与基础环境配置

- 提供 VS Code 安装服务（macOS / Windows 视客户终端而定）。
- 基础配置：插件市场访问策略、必要组件安装、常见冲突排查。
- **排除网络阻碍**：根据现场网络策略（如代理/白名单/SSL 检查等）给出可执行配置方案，确保 VS Code 能正常工作。

### 3.2 GitHub 账号注册与可用性保障

- 提供 GitHub 账号注册协助（含注册流程指导、必要的验证环节支持）。
- 验证账号登录与基本安全设置（如 2FA 建议、恢复码留存建议）。

### 3.3 电信定制 V Kard Extension（Proxy）交付

- 提供电信定制 **V Kard Extension** 安装与配置。
- V Kard Extension 的定位：
  - 作为 **Proxy** 能力，使 VS Code 在受限网络环境下可正常访问所需服务。
  - 强化 Git 同步的稳定性与准确性（如 clone/pull/push 等常用操作链路稳定）。

> 注：V Kard Extension 的具体能力边界以电信侧定制规格为准；本文档描述其目标效果与交付责任。

## 4. 交付流程（建议）

1. **前置确认**：终端操作系统、管理员权限、网络策略（是否存在代理/审计/SSL 检查/白名单）。
2. **安装与配置**：安装 VS Code → 配置网络与基础组件 → 安装并配置 V Kard Extension。
3. **账号开通**：协助注册/登录 GitHub 账号，并完成基础可用性验证。
4. **验收测试**：在 VS Code 内完成 Git 操作验证（见第 5 节）。
5. **交付回执**：形成交付清单（安装版本、配置项、验证结果）。

## 5. 验收标准（Pass/Fail）

满足以下验收项即视为通过：

- **VS Code 可用**：客户端可正常启动，基础编辑/终端功能可用。
- **网络链路可用**：在客户网络环境下，VS Code 可通过既定策略访问所需服务（含代理策略）。
- **GitHub 账号可用**：可正常登录，账号状态正常。
- **Git 同步稳定**：在 VS Code 内完成一次完整链路：
  - `git clone`（或打开已有仓库）
  - `git pull`
  - 本地变更提交
  - `git push`

如上述任一关键项无法完成，则需要定位原因并给出处理建议/整改项后再验收。
