# TeamsCamp：TAES 与 MAR 的商业架构洞见

> **核心命题**：TAES 和 MAR 不是同一个 Mission，需要分离到各自的 Repo。

---

## 1. 元洞见：产品 vs 履约

| 公式 | 定位 | 本质 |
|------|------|------|
| **TAES** | 产品 OS（组织生产力操作系统） | 可复制的能力 |
| **MAR** | 履约实例（单使命仓库执行内核） | 可验收的结果 |

**类比**：
- TAES = iOS（操作系统）
- MAR = 一个 App 项目（在 iOS 上运行的具体应用）

## 3. 商业模式对比

### TAES = 产品公司

```
┌──────────────────────────────────────────────────────────────────┐
│  TAES = 产品公司                                                  │
│  ─────────────                                                   │
│  收入 = Σ (E队 × A托举订阅)                                       │
│  成本 = 框架研发 + 模板维护 + 品控运营                            │
│  利润 = 规模效应（边际成本趋零）                                   │
└──────────────────────────────────────────────────────────────────┘
```

### MAR = 履约项目

```
┌──────────────────────────────────────────────────────────────────┐
│  MAR = 履约项目                                                   │
│  ─────────────                                                   │
│  收入 = Mission 产出价值（E系数 × Baseline）                       │
│  成本 = Token 消耗 + 人工微调                                     │
│  价值 = 可验收交付物 + 证据链沉淀                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 4. 核心维度对比

| 维度 | TAES Repo | MAR Repo |
|------|-----------|----------|
| **主体** | TeamsCamp (T营) | EdgeTeam (E队) |
| **财务定位** | 收入中心 | 成本中心 |
| **销售物** | A 托举 = Workplane × AITa | Mission 产出 |
| **采购方** | E队 | （无，E队 是消费者） |
| **成功标准** | E队 增长 × 复购率 | E系数 × Mission 完成率 |
| **规模路径** | 更多 E队 使用同一框架 | 单 E队 更多 Mission 实例 |

---

## 5. 公式映射

### TAES 视角（宏观）

$$\text{TeamsCamp Revenue} = \sum_{i=1}^{n} (\text{E队}_i \times \text{A托举订阅}_i)$$

- **T营 持有**：Workplane + AITa（基础设施）
- **T营 销售**：MAR 模板 + A 托举能力
- **T营 服务**：多个 E队 的订阅与支持

### MAR 视角（微观）

$$E = M \times C^2 \times AI$$

- **E队 消费**：A 托举能力
- **E队 产出**：Mission 交付物 + 证据链
- **E队 沉淀**：E系数 + 飞轮加速

### EdgeTeam 的双重身份

**重要澄清**：EdgeTeam 在当前仓库中是**实体**，不是软件：

| 维度 | 说明 | 位置 |
|------|------|------|
| **作为 TAES 实体** | E队是 TAES 生态中的"操作单元" | `CONTEXT/Parties/EdgeTeam/` |
| **作为产品定义** | E队的 Playbook（生命周期、托举关系、毕业目标） | `CONTEXT/Products/EdgeTeam-playbook.md` |
| **作为 MAR 主体** | E队是 MAR 实例仓库的 Owner | `Teams/` 产出归属于 E队 |

E队不是软件，而是**被软件服务的主体**：
- 在 TAES 层面：E队 被 TeamsEdge 管理
- 在 MAR 层面：E队 可能使用 E卡软件（如果定义）
