# POP（站点资产）

> **全称**：Point of Presence
>
> **定位**：TAES 生态中的网络接入点，是托举效应"bit 可靠"的物质基础。

---

## 1. 定义

**POP** 是 T营 持有并服务 E队 的网络站点资产，包括：
- 物理/虚拟网络接入点
- Workplane 的载体（代理、VPN、API 通道）
- 跨境合规与稳定性保障
- 地理分布与冗余

---

## 2. 核心能力

| 能力 | 说明 |
|------|------|
| **网络接入** | 提供稳定的 AI 服务访问通道 |
| **跨境合规** | 满足中国互联网环境下的合规要求 |
| **冗余容灾** | 多站点部署，单点故障不影响可用性 |
| **带宽保障** | SLA 保障的网络质量 |

---

## 3. 与 Workplane 的关系

```
POP (物理载体)
    │
    └── 承载 ──→ Workplane (逻辑抽象)
                    │
                    └── 服务 ──→ E队 (网络可达)
```

> **Workplane** 是逻辑概念（"工作平面"），**POP** 是物理实现（"接入点"）。

---

## 4. 站点类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **云端 POP** | 部署在公有云的虚拟接入点 | Azure HK / AWS Tokyo |
| **边缘 POP** | 部署在边缘节点的物理接入点 | 香港机房、东京机房 |
| **混合 POP** | 云端 + 边缘混合部署 | 主用云端，备用边缘 |

---

## 5. 审计字段（最小集）

| 字段 | 类型 | 说明 |
|------|------|------|
| `pop_id` | string | 站点唯一标识 |
| `location` | string | 地理位置 |
| `type` | enum | cloud / edge / hybrid |
| `status` | enum | active / maintenance / offline |
| `sla_uptime` | decimal | SLA 可用性承诺（如 99.9%） |
| `served_et_codes` | array | 服务的 E队 列表 |

---

## 6. 边界（Limit）

- POP **由 T营 持有**，E队 通过 Workplane 使用
- POP **不包含** 算力资源（归 AITC）
- POP **不处理** 身份认证（归 NexusPass）
