## 1) 双A策略：Augment × Allied

### 核心定位

```
┌─────────────────────────────────────────────────────────────────┐
│                        双A策略                                   │
│                                                                 │
│    ┌──────────────────┐         ┌──────────────────┐           │
│    │    Augment       │    ×    │     Allied       │           │
│    │    托举           │         │     协同          │           │
│    │   ─────────      │         │   ─────────      │           │
│    │   业务锚点        │         │   品牌锚点        │           │
│    │   方法论层面      │         │   客户感知层面     │           │
│    │   "我们做什么"    │         │   "客户感受什么"   │           │
│    └──────────────────┘         └──────────────────┘           │
│                     ↓                    ↓                      │
│              ┌─────────────────────────────────────┐            │
│              │  T营 托举(Augment) E队，            │            │
│              │  E队 获得协同(Allied) AI 能力       │            │
│              └─────────────────────────────────────┘            │
└─────────────────────────────────────────────────────────────────┘
```

### Augment（托举）— 业务锚点

> **定义**：T营 通过基础设施让 E队 获得"原本不可及"的 AI 能力。

| 维度 | 说明 |
|:---|:---|
| **词性** | 动词 → 名词化 |
| **语义** | 增强、放大、托举 |
| **使用场景** | 方法论文档、技术架构、内部沟通 |
| **核心公式** | `托举效应 = 网络可靠(bit) × AI可用(Token)` |

**Augment 的三层含义**：

1. **网络托举**（Workplane）— 让 E队 稳定合规地访问全球 AI
2. **算力托举**（AITa）— 让 E队 使用顶级 AI 能力而无需自建
3. **方法托举**（TAES）— 让 E队 用可验收的方式交付 AI 产出

### Allied（协同）— 品牌锚点

> **定义**：与 E队 并肩作战的 AI 伙伴，而非取代人的工具。

| 维度 | 说明 |
|:---|:---|
| **词性** | 形容词 → 名词化 |
| **语义** | 联盟、盟友、协同、并肩 |
| **使用场景** | 品牌传播、客户沟通、产品命名 |
| **情感调性** | 温暖、信任、共同作战 |

**为什么选择 Allied 而非 Agentic？**

| 对比 | Allied | Agentic |
|:---|:---|:---|
| **语义** | 盟友、协同 | 代理、自主 |
| **情感** | 温暖、可信赖 | 技术、冷酷 |
| **关系** | 人机并肩 | AI 独立运行 |
| **适用** | ToB 企业客户 | ToD 开发者 |

> **品牌决策**：TAES 面向 ToB 企业，"协同"比"自主"更让客户安心。

### 双A的协同表达

## 6) Allied AI 品牌定义

### Allied AI 是什么？

> **Allied AI** = 在特定 Mission 上，产出质量和效率达到或超越同岗位人类平均表现的 AI。

### Allied AI 的品质承诺

| 维度 | 承诺 | 验证方式 |
|:---|:---|:---|
| **质量** | Mission 验收通过率 ≥ 人类基线 | E队 验收 |
| **效率** | 完成时间 ≤ 人类基线 × 0.5 | 时间戳记录 |
| **可靠** | 7×24 可用，SLA 保障 | Workplane 监控 |
| **合规** | 数据不出境，审计可追溯 | Evidence 链 |

### Allied 的三层含义

```
Allied（协同）
  │
  ├── Allied Forces（联盟军）
  │     → 人 + AI 是并肩作战的战友
  │
  ├── Allied Quality（盟友品质）
  │     → AI 产出达到"可信赖队友"的标准
  │
  └── Allied Spirit（协同精神）
        → 人负责判断，AI 负责执行
```

### 协同分工原则

| 角色 | 职责 |
|:---|:---|
| 👤 **人负责** | 意图定义、边界判断、验收决策 |
| 🤖 **AI负责** | 批量执行、模式识别、草稿生成 |
| 🤝 **协同价值** | 人的判断力 × AI 的执行力 |

## 9) 检查清单

### 对外材料发布前检查

- [ ] **双A出现**：材料中是否体现 Augment（托举）和 Allied（协同）？
- [ ] **术语合规**：是否使用了内部术语（AITa/AITC）？如有，需替换
