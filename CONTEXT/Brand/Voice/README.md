## 品牌语调 (Tone of Voice)

### 总体原则

| 维度 | 是 | 不是 |
|------|-----|------|
| **专业度** | 专业、可信 | 学术、晦涩 |
| **温度** | 友好、平等 | 冷漠、居高临下 |
| **节奏** | 简洁、有力 | 冗长、啰嗦 |
| **立场** | 自信、明确 | 模糊、犹豫 |

### 各品牌语调差异

| 品牌 | 语调特征 | 示例 |
|------|----------|------|
| **TeamsEdge** | 权威、专业 | "AI-Native 是组织能力，不是工具清单" |
| **TeamsCamp** | 温暖、支持 | "T营 托举 E队，让专业的人做专业的事" |
| **EdgeTeam** | 活力、行动 | "一个任务，一个 Repo，搞定" |
| **Allied AI** | 启发、引导 | "自然言語でコードを紡ぐ" |

## 术语规范 (Terminology)

### 必须使用的术语

| 术语 | 定义 | 禁止替换为 |
|------|------|-----------|
| **祘力** | 基于微软订阅的 AI 算力 | "算力"、"计算资源" |
| **E队** | EdgeTeam 操作单元 | "团队"、"小组" |
| **T营** | TeamsCamp 业务主体 | "公司"、"组织" |
| **托举** | T营 对 E队 的支持模式 | "支持"、"赋能" |
| **Mission** | 任务 (MR 核心概念) | "项目"、"需求" |

### 术语使用场景

```markdown
✅ 正确:
"E队 T189 通过 AITC 获得祘力托举"
"这个 Mission 需要 3 个 E队 协作"

❌ 错误:
"团队 T189 通过算力资源获得支持"
"这个项目需要 3 个小组协作"
```

## 写作规范

### 标题

- 使用动词开头："创建"、"配置"、"定义"
- 避免问句式标题
- 层级不超过 4 级 (H1-H4)

### 正文

- 段落不超过 5 行
- 使用列表拆分复杂内容
- 技术术语首次出现时给出定义

### 代码与命令

- 代码块标注语言
- 命令给出预期输出
- 危险操作标注 ⚠️ 警告
