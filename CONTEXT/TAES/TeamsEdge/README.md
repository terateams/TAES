## 战略定位

### 我们解决什么问题？

**E队（客户团队）面临两个核心障碍：**

1. **到不了** — 物理/合规上无法访问全球 AI 服务
2. **用不起** — 经济/技术上无法负担顶级 AI 能力

**T营（協力營）的价值主张：**

> 通过 **托举效应**，让 E队 在有限预算内获得"原本不可及"的 AI 能力，  
> 使 **人的判断力 × AI 的执行力** 成为可能。

### 商业模式一句话

```
T营 持有基础设施 → 托举 E队 使用 AI → E队 产出 Mission 交付物 → 价值闭环
```

## 软件定位：元协力软件

### "元协力软件"四维定义

| 维度 | 关键字 | 内涵 | 具体表现 |
| :---: | :---: | :--- | :--- |
| **元** | 源代码 | 以 Repo 为载体，CI/CD 工作流 | DevOps 模式，代码即配置，版本可追溯 |
| **协** | Allied | 标准 BOSS 业务支撑 + Allied AI 维度 | VSC + TeamsEdge 双轮驱动，人机协作 |
| **力** | AITa | "费米能级"以上的 LLM 算力 | 季度确认、付费采购、能力持续迭代 |
| **软件** | 动态主体 | 软件本身是动态生长的新物种 | 以番号 + 域名的形式存在，持续演进 |

> **一句话**：TeamsEdge 是源代码形态、Allied 协同驱动、AITa 算力加持、动态生长的新一代业务支撑软件。

### 关键概念解读

**元（Meta）**
- 软件以**源代码**形式存在于 Repo 中
- 遵循 **CI/CD** 工作流，所谓 DevOps
- 配置即代码，变更可审计，历史可回溯

**协（Allied）**
- 在标准 **BOSS（Business & Operation Support System）** 业务支撑范畴内
- 增加 **Allied AI** 维度：VSCode + GitHub Copilot + TeamsEdge 三位一体
- **Allied** = 人与 AI 的协作界面，不是 AI 替代人，而是 AI 托举人
- 协同价值 = 人的判断力 × AI 的执行力

**力（AITa）**
- **AITa** = AI 算力的品牌符号（AI + Ta），核心生产资料
- 需达到 **"费米能级"** 以上的 LLM 能力（Frontier Model）
- 每季度进行 AITa 能力确认与更新
- 付费采购，确保服务稳定与能力持续
- 当前参考：Claude Opus / GPT-4o / Gemini Ultra

**软件（Software）**
- 软件本身是**动态生长**的有机体
- 以 **番号 + 域名** 的形式存在（如 `ct3.lsn189.cn`）
- 不是静态产品，而是持续演进的服务载体
