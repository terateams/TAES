## 各文件夹详解

### 01-inbox/ — 快速捕捉区

**用途**：所有临时内容的入口，不需要思考分类

| 子文件夹 | 存放内容 |
|---------|---------|
| `screenshots/` | 截图、参考图片 |
| `clips/` | 复制粘贴的文字、链接 |
| `quick-asks/` | 快速问 AI 的记录 |

**管理规则**：
- 每周清理一次（建议周五下午）
- 有价值的 → 移到对应位置
- 没价值的 → 删除
- 目标：inbox 永远不超过 20 个文件

---

### 02-projects/ — 项目工作区

**用途**：当前进行中的商业项目

**项目文件夹结构**：
```
[项目名]/
├── brief.md           # 项目简述（目标、背景、关键人）
├── drafts/            # 草稿迭代（v1, v2, v3...）
├── ai-sessions/       # AI 对话记录
│   ├── opus-策略讨论.md
│   ├── gemini-数据分析.md
│   └── gpt-文案撰写.md
└── final/             # 最终交付物
```

**`_templates/` 文件夹**：存放可复用的项目模板

---

### 03-prompts/ — 提示词库（核心资产）

**用途**：积累有效的 AI 提示词，这是你最宝贵的资产

**INDEX.md 快速查找**：
```markdown
## 按场景查找
- 写邮件 → by-task/writing/email.md
- 做分析 → by-task/analysis/
- 说服人 → by-task/communication/persuasion.md
- 定战略 → by-task/strategy/

## 按模型查找
- 深度思考 → by-model/opus-best.md
- 数据处理 → by-model/gemini-best.md
- 创意文案 → by-model/gpt-best.md
```

---

### 04-knowledge/ — 知识沉淀

| 子文件夹 | 内容 |
|---------|------|
| `insights/` | 从 AI 对话中提炼的商业洞见 |

| `model-learnings/` | 各模型使用心得 |

### 05-cases/ — 问题解决档案

**用途**：记录"问题 → 过程 → 答案"的完整路径

**按月归档**：`YYYY-MM/[主题].md`

**价值**：下次遇到类似问题，先搜这里

---

**用途**：已完成的项目整体移入

**规则**：项目完成后，整个文件夹从 `02-projects/` 移到这里

### _tools/ — 辅助脚本

**用途**：存放 Copilot 帮你写的简单自动化脚本

## 模型选择策略

### 模型特性对照表

| 模型 | 最擅长 | 适用场景 |
|------|--------|---------|
| **Opus 4.5** | 深度思考、复杂推理、平衡观点 | 战略分析、重要决策、敏感沟通 |
| **Gemini 3.0 Pro** | 数据处理、长上下文、多模态 | 报表分析、文档整理、图片理解 |
| **ChatGPT 5.2** | 创意发散、流畅文字、快速响应 | 文案撰写、头脑风暴、日常沟通 |

### 任务-模型匹配指南

| 任务类型 | 首选 | 备选 |
|---------|------|------|
| 战略规划 | Opus | - |
| 市场分析 | Opus | Gemini（如有大量数据）|
| 财务分析 | Gemini | Opus |
| 数据整理 | Gemini | - |
| 营销文案 | GPT | - |
| 邮件撰写 | GPT | Opus（重要邮件）|
| 头脑风暴 | GPT | - |
| 风险评估 | Opus | - |
| 会议纪要 | Gemini | GPT |
| 演讲稿 | GPT | Opus（正式场合）|
| 重要决策 | **三个都问** | 比较后综合判断 |

### 多模型协作流程

```
1. 问题定义（自己写清楚）
        ↓
2. 快速探索（GPT 5.2）
   → "帮我发散一下这个问题的可能角度"
        ↓
3. 深度分析（Opus 4.5）
   → "请从战略角度深入分析..."
        ↓
4. 数据验证（Gemini 3.0 Pro）
   → "请整理这些数据，验证上述假设"
        ↓
5. 综合判断（自己）
   → 比较三个模型的输出，形成最终观点
```

## 工作流程

### 日常工作流

```
早上：
└── 检查 inbox，快速处理或分类

工作中：
├── 临时内容 → inbox/
├── 项目工作 → projects/[项目名]/
├── 有效提示词 → prompts/
└── 有价值洞见 → knowledge/

周五：
├── 清理 inbox
├── 更新 prompts/INDEX.md
└── 归档本周完成的项目
```

### 项目工作流

```
启动：
├── 从 _templates/ 复制模板
├── 填写 brief.md
└── 在 projects/ 建立项目文件夹

进行中：
├── 草稿存入 drafts/（v1, v2...）
├── AI 对话记录存入 ai-sessions/
└── 有效提示词同步到 prompts/

完成：
├── 最终版存入 final/
├── 整个项目移到 archive/
└── 写一条 case 记录（如有价值）
```

### 学习积累流

```
每次 AI 对话后问自己：
├── 这个提示词有效吗？ → prompts/
├── 学到新洞见了吗？ → knowledge/insights/
├── 发现模型特性了吗？ → knowledge/model-learnings/
└── 失败了？为什么？ → knowledge/failures/
```

## 额度使用建议

3000 次/月 ≈ 100 次/天

| 用途 | 建议占比 | 约次数/天 |
|------|---------|----------|
| 日常快速问答 | 40% | 40 |
| 项目深度工作 | 40% | 40 |
| 实验/比较模型 | 20% | 20 |

**原则**：不要省着用，但要有记录

---

## 核心理念

> **你的 3000 次/月 真正的价值，不是用掉它，而是把用它学到的东西沉淀下来。**

三位专家的共识：
1. **用起来** — 大量实践才能理解 AI 边界
2. **记下来** — 提示词和心得是核心资产
3. **理解它** — 知道为什么有效比知道有效更重要
4. **保持简单** — 最好的系统是你真正会用的系统
