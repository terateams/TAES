# TeamsCamp（T营）— 业务主体（Principal）

> **定位**：TAES 生态中的"资源持有方 / 业务主体"。
>
> 一句话：**T营是持有基础设施与合规责任的经营主体，通过托举效应赋能 E队。**

---

## 1. 定义

**TeamsCamp（T营 / 協力營）** 是 TAES 中的主体（Principal）：
- 在组织关系中：T营是 **服务提供方**（持有算力、网络、身份等基础设施）。
- 在合规关系中：T营是 **责任承担方**（承担法律、财务、数据合规责任）。
- 在商业关系中：T营是 **价值分发方**（将能力授权给 E队 使用）。

---

## 2. 核心职责

| 职责 | 说明 |
|------|------|
| **资源持有** | 持有 AITC（算力）、POP（站点）等基础设施 |
| **能力分发** | 将资源授权给 E队 使用 |
| **合规保障** | 承担法律主体、数据合规、财务审计责任 |
| **身份管理** | 管理 NexusPass 身份体系 |
| **运营支撑** | 通过 TeamsEdge 系统管理 E队 生命周期 |

---

## 3. 持有的资源

| 资源 | 类型 | 说明 |
|------|------|------|
| **AITC** | 算力资产 | AI 集成租户算力（Azure/Google/Anthropic） |
| **POP** | 站点资产 | 网络接入点，Workplane 的物理载体 |
| **NexusPass** | 身份服务 | 跨系统认证与权限管理 |
| **域名资产** | 品牌资产 | 顶级域名与 SSL 证书 |

---

## 4. 与 E队 的关系

```
T营 (持有资源)
    │
    │ 托举效应 = Workplane (bit可靠) × AITa (Token可用)
    │
    ↓
E队 (消费资源、执行 Mission、产出交付物)
```

> **核心洞见**：托举效应是**乘法关系**，缺一归零。T营的价值是确保这两项同时可用。

---

## 5. 边界（Limit）

- T营 **不执行** 具体 Mission（执行归 E队）
- T营 **不拥有** E队 的交付物（交付物归 E队）
- T营 **不替代** E队 做验收决策（验收责任归 E队）

---

## 6. 与 TAES 的映射

| TAES 要素 | 对应 | 说明 |
|-----------|------|------|
| **T** | TeamsCamp | 本文档定义的主体 |
| **A** | Augment | T营持有的基础设施构成托举效应 |
