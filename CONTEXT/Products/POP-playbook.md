## 0. 概述 (Overview)

> 🌐 **POP** = Point of Presence，TeamsEdge 生态的物理/云化站点资产，提供计算、存储、网络的基础设施托举。

**POP** 是 T营 持有的站点资产，包括物理机房、边缘节点、云资源池，与 AITC（软件订阅资产）共同构成赋能基础设施的双支柱：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                  赋能基础设施 (Enabled Infra) 双支柱                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌───────────────────────────────┐   ┌───────────────────────────────┐     │
│  │           AITC                │   │            POP                │     │
│  │      软件订阅资产              │   │        站点资产               │     │
│  │   (AI TeamsCamp)              │   │   (Point of Presence)         │     │
│  ├───────────────────────────────┤   ├───────────────────────────────┤     │
│  │  • M365 EDU/NGO/BIZ           │   │  • 物理机房 (Colo)            │     │
│  │  • 席位 (License)             │   │  • 边缘节点 (Edge)            │     │
│  │  • 租户 (Tenant)              │   │  • 云资源池 (Cloud)           │     │
│  │  • AI-ready 工具链            │   │  • 计算/存储/网络             │     │
│  └───────────────────────────────┘   └───────────────────────────────┘     │
│                    │                               │                        │
│                    └───────────┬───────────────────┘                        │
│                                │                                            │
│                                ▼                                            │
│                    TC (T营) 持有 → ET (E队) 使用                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 4. 托举效应分析 (Hosting Effect)

### 4.1 E队 自建 vs POP 托举

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      POP 托举效应对比                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  E队 自行采购                        POP 托举效应                            │
│  ───────────────────────────────────────────────────────────────────────── │
│  单独签约云商 (无折扣)               TC 批量账户 (EA/CSP 折扣) ⭐             │
│  个人/小企业账户                     企业认证账户 (更高配额) ⭐               │
│  自行运维                            专业运维团队 ⭐                         │

│                                                                             │
│  托举效应:                                                                   │
│  • 价格优势: EA/CSP 批量折扣、长期合约优惠                                   │
│  • 资质优势: 企业账户配额、ICP 备案关联                                      │
│  • 运维优势: 专业团队、统一监控、7×24 响应                                   │

```

### 4.2 托举效应等级

| 等级 | 含义 | 示例 |
| :--- | :--- | :--- |
| ⭐⭐⭐ | 显著优势，E队 难以独立获得 | Colo 机房、Azure EA、Cloudflare Enterprise |

## 11. 成本管理 (Cost Management)

### 11.1 成本分类

| 成本类型 | 说明 | 计费模式 | 示例 |
| :--- | :--- | :--- | :--- |
| **机房租赁** | Colo 机柜/U位租金 | 月租/年租 | POP-TYO-CC1 |
| **电力** | 机房电费 | 按用量 | kWh 计费 |
| **带宽** | 网络流量费用 | 按用量/包月 | Mbps/GB |
| **云资源** | IaaS/PaaS 费用 | 按用量/预留 | vCPU·小时 |
| **许可证** | 软件授权费用 | 订阅 | Cloudflare Pro |

### 11.2 成本优化策略

| 策略 | 说明 | 适用场景 |
| :--- | :--- | :--- |
| **预留实例** | 1-3年预付获取折扣 | 稳定负载云资源 |
| **Spot/抢占式** | 竞价实例，大幅折扣 | 可中断任务 |
| **区域优化** | 选择成本较低区域 | 对延迟不敏感场景 |
| **资源整合** | 合并小规格为大规格 | 多个小 VM |
| **自动伸缩** | 按需扩缩容 | 突发负载 |
