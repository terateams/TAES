## 0. 概述 (Overview)

> 🏛️ **核心定义**: T营 是 TeamsEdge 生态的 **业务主体 (Business Entity)**，具备托举能力，通过分派 **赋能基础设施 (Enabled Infra)** 为 E队 提供 **起步空间 (Launchpad)**。

**TeamsCamp (T营 / TC)** 不仅仅是"公司"，而是具备完整托举能力的业务主体，为 E队 提供法律、支付、合规、服务四个维度的赋能基础设施支撑，助力 E队 成长直至毕业。

```

## 2. 核心原则 (Principles)

> 🏛️ **设计理念**: T营 是 E队 的业务主体载体，提供赋能基础设施，助力 E队 成长毕业。

| # | 原则 | 说明 |
| :--- | :--- | :--- |
| **TCP-1** | **完整能力** | T营 必须具备法律、支付、合规、服务四个维度的完整能力 |
| **TCP-2** | **差异定位** | 每个 T营 有明确的差异化定位和专长领域 |
| **TCP-3** | **互补协作** | T营 之间互补协作，覆盖全场景需求 |
| **TCP-4** | **T1 兜底** | TC1 (万联) 作为终极兜底，处理其他 T营 无法覆盖的场景 |
| **TCP-5** | **NP 锚定** | 每个 T营 至少有一个 Don 级别的 NP |
| **TCP-6** | **毕业导向** | 托举是阶段性的，目标是助力 E队 毕业独立 |
| **TCP-7** | **赋能优先** | 提供有优势的资源 (Scale/Benefit)，而非普通商品化资源 |

## 5. 赋能基础设施 (Enabled Infra)

### 5.1 赋能 vs 普通基础设施

> 🎯 **核心理念**: T营 提供的不是普通商品化资源，而是具有**优势**的赋能基础设施。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                Enabled Infra vs. Commodity Infra                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  普通基础设施 (Commodity)          赋能基础设施 (Enabled)                    │
│  ─────────────────────────────────────────────────────────────────────────  │
│  M365 Business (商业定价)          M365 Education (教育免费/低价) ⭐         │
│  M365 Business (商业定价)          M365 NGO (非营利优惠) ⭐                  │
│  阿里云按需付费                    阿里云教育优惠/创业扶持 ⭐                 │
│  普通域名注册                      品牌域名组合 (ilya.team 生态)             │
│  单一云账户                        多云冗余架构                              │
│  普通信用卡支付                    国际支付通道 (T6 能力)                    │
│                                                                             │
│  赋能 = Scale优势 + Benefit优势 + 生态协同 + 专业运维                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 赋能的三个层次

| 层次 | 含义 | 示例 |
| :--- | :--- | :--- |
| **价格赋能** | 低于市场价获取资源 | M365-EDU 免费席位、NGO 折扣、创业扶持 |
| **能力赋能** | 单一 E队 无法获得的能力 | 国际支付通道、ICP 备案资质、专业运维 |
| **风险赋能** | 降低运营风险 | 多 T营 协作、备援链、合规兜底 |

## 6. 毕业机制 (Graduation)

### 6.1 毕业的定义

> 🎓 **毕业**: E队 达到独立运营能力，不再需要 T营 的核心托举服务。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     E队 生命周期与毕业                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐                  │
│  │ 孵化期  │───►│ 成长期  │───►│ 成熟期  │───►│ 毕业    │                  │
│  │ Incubate│    │ Growth  │    │ Mature  │    │ Graduate│                  │
│  └─────────┘    └─────────┘    └─────────┘    └─────────┘                  │
│       │              │              │              │                        │
│       ▼              ▼              ▼              ▼                        │
│   全托举          部分托举       能力转移       独立运营                      │
│   (依赖T营)      (协作T营)     (自建能力)    (校友网络)                      │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.2 毕业条件

| 维度 | 条件 | 说明 |
| :--- | :--- | :--- |
| **AI Ready** | 具备 AI 工具链使用能力 | 能独立使用 AI 辅助工作流 |
| **Network Ready** | 具备独立网络基础设施 | 自有域名、云账户、支付能力 |
| **Legal Ready** | (可选) 具备独立法人实体 | 有自己的公司或组织 |

### 6.3 毕业后关系

| 类型 | 说明 | 与 T营 关系 |
| :--- | :--- | :--- |
| **升级为 T营** | 规模足够大，成为新的托举主体 | 平级协作 |
| **校友 E队** | 保持松散协作关系 | Alumni Network |
| **完全独立** | 彻底脱离生态 | 无 |

### 6.4 毕业不等于断联

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     毕业后的价值循环                                          │
├─────────────────────────────────────────────────────────────────────────────┤

│  毕业 E队                          T营 / 新 E队                              │
│  ┌─────────────────┐               ┌─────────────────┐                     │
│  │  Alumni E队     │               │  T营 / 孵化E队   │                     │
│  │                 │ ──反向服务──► │                 │                     │
│  │  • 经验分享     │               │  • 接受服务     │                     │
│  │  • 商业合作     │               │  • 业务对接     │                     │
│  │  • 资源互换     │ ◄──生态协同── │  • 新项目孵化   │                     │
│  │                 │               │                 │                     │
│  └─────────────────┘               └─────────────────┘                     │
│                                                                             │
