## 0. 概述 (Overview)

> 🚀 **核心定义**: E队 (EdgeTeam/协力营) 是 TeamsEdge 生态的 **操作单元 (Operating Unit)**，被 T营 托举，目标是成长并最终 **毕业 (Graduation)**。

**EdgeTeam (E队/协力营)** 是具有独立使命的操作单元，在 T营 的托举下获得赋能基础设施支撑，完成从孵化到毕业的成长旅程。

## 2. 核心原则 (Principles)

> 🚀 **设计理念**: E队 是为成长而生的操作单元，托举是阶段性的，毕业是终极目标。

| # | 原则 | 说明 |
| :--- | :--- | :--- |
| **ET-1** | **独立使命** | 每个 E队 必须有清晰的使命和存在价值 |
| **ET-2** | **一号一队** | 每个 E队 有且仅有一个番号，番号是思维锚点 |
| **ET-3** | **必有托举** | E队 必须有至少一个 T营 作为主托举方 |
| **ET-4** | **Don 负责** | 每个 E队 必须有一个 Don 作为负责人 |
| **ET-5** | **毕业导向** | 托举是阶段性的，E队 应追求毕业独立 |
| **ET-6** | **价值创造** | E队 存在的目的是创造价值，而非消耗资源 |
| **ET-7** | **能力成长** | E队 应在托举过程中持续提升自身能力 |
| **ET-8** | **Mission 驱动** | E队 的一切工作以 Mission 为驱动单元，对齐 TAES 方法论 |
| **ET-9** | **AI 贡献度** | E队 成熟度以 AI Contribution Rate 为核心度量指标 |

## 3. Mission 驱动工作方式 (Mission-Driven)

> 🚀 **核心理念**: E队 的工作以 Mission 为最小驱动单元，每个 Mission 需满足 TAES 五要素。

### 3.1 Mission 五要素

基于 [TAES 方法论](../../TAES.md)，E队 的每个 Mission 必须具备以下五要素：

| # | 要素 | 英文 | 说明 | 示例 |
| :--- | :--- | :--- | :--- | :--- |
| **M-1** | 明确输入 | Clear Input | Mission 的起点和触发条件清晰 | 客户需求文档、会议纪要 |
| **M-2** | 预期输出 | Expected Output | 交付物有清晰定义 | 设计稿、代码、报告 |
| **M-3** | 验收标准 | Acceptance Criteria | 可验证的完成标准 | Checklist、测试用例 |
| **M-4** | AI 可介入 | AI-Assistable | 任务可被 AI 工具辅助执行 | Copilot 可生成、可审查 |
| **M-5** | 范围约束 | Scope Constraint | 任务边界明确，防止蔓延 | 时间框、功能边界 |

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Mission 五要素验证模型                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────┐      ┌─────────────┐      ┌─────────────┐                 │
│  │  明确输入    │──────│   Mission   │──────│  预期输出   │                 │
│  │  (Input)    │      │   执行体    │      │  (Output)   │                 │
│  └─────────────┘      └─────────────┘      └─────────────┘                 │
│         │                    │                    │                         │
│         ▼                    ▼                    ▼                         │
│  ┌─────────────┐      ┌─────────────┐      ┌─────────────┐                 │
│  │ AI 可介入   │      │ 范围约束    │      │ 验收标准    │                 │
│  │ (AI-Assist) │      │  (Scope)    │      │(Acceptance) │                 │
│  └─────────────┘      └─────────────┘      └─────────────┘                 │
│                                                                             │
│  验证规则: 五要素缺一不可，否则 Mission 不应启动                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 Mission 状态流转

| 状态 | 说明 | AI 参与度 |
| :--- | :--- | :--- |
| **Draft** | Mission 草案，五要素不完整 | AI 辅助补全要素 |
| **Ready** | 五要素完备，可启动 | AI 可介入执行 |
| **In Progress** | 执行中 | AI 持续辅助 |
| **Review** | 等待验收 | AI 辅助审查 |
| **Done** | 完成并归档 | AI 生成总结 |

### 3.3 ICE 作业框架 (Intent-Constraints-Evidence)

基于 TAES 方法论，E队 成员与 AI 协作时应采用 **ICE 框架**：

| 要素 | 英文 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| **意图** | Intent | 明确告诉 AI 想要完成什么 | "帮我生成一个 API 接口文档" |
| **约束** | Constraints | 定义边界、标准和限制条件 | "使用 OpenAPI 3.0 格式，中文描述" |
| **证据** | Evidence | 提供上下文、参考和验证依据 | "参考现有 User API 的格式" |

```

│                         ICE 作业框架                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│                    ┌─────────────────┐                                    │
│                    │  Intent (意图)  │                                    │
│                    │  "我要做什么"   │                                    │
│                    └────────┬────────┘                                    │
│                             │                                              │
│              ┌─────────────┴─────────────┐                              │
│              ▼                             ▼                              │
│    ┌─────────────────┐         ┌─────────────────┐                    │
│    │ Constraints     │         │    Evidence    │                    │
│    │  (约束)          │         │    (证据)       │                    │
│    │ "边界和标准"   │         │ "上下文与参考" │                    │
│    └─────────────────┘         └─────────────────┘                    │
│              │                             │                              │
│              └─────────────┬─────────────┘                              │
│                            ▼                                               │
│                   ┌─────────────────┐                                   │
│                   │   AI 输出结果   │                                   │
│                   │  可验证、可追溯  │                                   │
│                   └─────────────────┘                                   │
│                                                                             │
│  💡 ICE 使用技巧:                                                           │
│  • Intent 要明确、具体，避免模糊描述                                         │
│  • Constraints 消除歧义，减少 AI “猜测”                                    │
│  • Evidence 提供足够上下文，让 AI 产出一致结果                             │
│                                                                             │

```

### 3.5 E队 指标体系 (TAES Layer 1-4)

基于 TAES 方法论，E队 成长采用四层指标体系，**Layer-4 AI 贡献度是核心指标**：

| Layer | 指标维度 | 英文 | 说明 | 关键指标 |
| :--- | :--- | :--- | :--- | :--- |
| **Layer-1** | 连接性 | Connectivity | Workplane 连接是否正常 | 登录率、工具使用率 |
| **Layer-2** | AI 执行质量 | AI Execution | AI 辅助产出的质量 | 采纳率、修改率 |
| **Layer-3** | 业务成果 | Business Outcome | 任务完成度和业务价值 | Mission 完成率、交付质量 |
| **Layer-4** | **AI 贡献度** | **AI Contribution** | **AI 在价值创造中的占比** | **AI Contribution Rate** |

## 10. 毕业详细流程 (Graduation Process)

### 10.1 毕业前准备清单

| # | 准备项 | 说明 | 验证方式 |
| :--- | :--- | :--- | :--- |
| **1** | 自有域名 | 已注册并完成转移 | DNS 配置验证 |
| **2** | 自有云账户 | 阿里云/AWS/Azure 等 | 账户截图 |
| **3** | 自有支付能力 | 银行账户/信用卡 | 支付验证 |
| **4** | 自有 AI 订阅 | M365 Copilot / GitHub Copilot 等 | 订阅截图 |
| **5** | 数据迁移完成 | 所有业务数据已迁移 | 迁移报告 |
| **6** | 费用结清 | 无欠款 | 财务确认 |

### 10.2 毕业交接流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         E队 毕业交接流程                                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Step 1: 毕业申请                                                            │
│  ├── E队 Don 提交毕业申请                                                   │
│  └── 附带自评报告 (AI Ready + Network Ready 证明)                           │
│                                                                             │
│  Step 2: 条件验证 (T营 Don)                                                  │
│  ├── 验证 AI Ready: 自有 AI 订阅能力                                        │
│  ├── 验证 Network Ready: 自有基础设施                                       │
│  └── 验证费用结清                                                           │
│                                                                             │
│  Step 3: 资源结算                                                            │
│  ├── 统计未归还资源                                                         │
│  ├── 计算结算金额                                                           │
│  └── 完成财务结算                                                           │
│                                                                             │
│  Step 4: 能力移交                                                            │
│  ├── NP 移交至 E队 (或新建独立 NP)                                          │
│  ├── 域名转移完成                                                           │
│  ├── 数据迁移完成                                                           │
│  └── AITC 资源回收                                                          │
│                                                                             │
│  Step 5: 状态变更                                                            │
│  ├── E队 状态 → Alumni                                                      │
│  ├── 番号永久保留 (不复用)                                                   │
│  └── 进入校友网络                                                           │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 10.3 毕业后权益

| 权益 | 说明 |
| :--- | :--- |

| **校友网络** | 加入 Alumni 网络，保持生态联系 |
| **优惠采购** | 部分资源可享受校友优惠价格 |
