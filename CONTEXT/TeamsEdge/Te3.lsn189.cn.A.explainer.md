## §1 深层洞见

### 洞见一：A 不是三个子模块，而是一条"托举链"

表面看，Workplane / AITa / AC 是三个并列支柱。深层来看，它们是一条**串联的托举链**，遵循严格的依赖顺序：

```
Workplane → AITa → AC → E队 可以完成 Mission
   ↓          ↓       ↓
  物理层    算力层   语义层
   ↓          ↓       ↓
 "到得了"  "调得动" "懂得透"
```

**深层含义**：
- **Workplane = 0** → 后面全部归零（网不通，AI 再强也无意义）
- **AITa = 0** → AC 形同虚设（有语境但没算力去理解）
- **AC = 0** → AI 变成通用助手（有算力但不懂你的业务）

> **洞见**：三者是**串行乘法**，不是**并行加法**。这解释了为什么 TAES 强调"托举效应 = 乘法关系，缺一归零"。

---

### 洞见二：A 是 E=MC²AI 公式中"被折叠的复杂度"

**A 折叠了什么复杂度？**

| 对外承诺（UI 呈现） | 对内实现（后端复杂度） |
|---------------------|------------------------|
| "bit 可靠" | 隧道协议、出口节点、路由策略、合规适配 |
| "Token 可用" | 多供应商订阅、席位分发、配额治理、账单结算 |
| "Context 可懂" | 语境建模、CA 协议、ACD 四象限、ACC 渲染 |

> **洞见**：A 的价值不是"提供三个功能"，而是**把复杂度折叠成简单承诺**——让 E队 只需关心 Mission。

---

### 洞见三：A 的三支柱对应三种"不可达"

| 失败模式 | 根因 | A 的对策 | 典型症状（用户可见） |
|----------|------|----------|----------------------|
| **物理不可达** | 网络封锁/不稳定 | Workplane | 连不上/超时/被墙/路由错误 |
| **经济不可达** | 成本/权限/配额 | AITa | 没账号/没配额/限流/欠费 |
| **语义不可达** | AI 不懂业务 | AC | 答非所问/幻觉/上下文缺失 |

**诊断优先级（UI 排障路径）**：
```
Step 1: Workplane OK?  → 不 OK → 显示网络错误提示
        ↓ OK
Step 2: AITa OK?       → 不 OK → 显示配额/权限提示
        ↓ OK
Step 3: AC OK?         → 不 OK → 显示语境缺失提示
        ↓ OK
Step 4: 问题在 Mission 本身
```

> **洞见**：A 的三支柱不是"功能分类"，而是**故障诊断树**——从物理层往语义层逐层排查。

---

### 洞见四：A 的权限边界揭示了"谁为什么负责"

| 支柱 | T营 | E队 | 边界本质 | UI 权限 |
|------|:---:|:---:|----------|---------|
| **Workplane** | 写 | 只读 | 物理资产 | E队：只看 Dashboard |
| **AITa** | 写 | 只读 | 算力契约 | E队：只看用量 |
| **AC** | 运维 | 贡献 | 语境共建 | E队：可编辑 Context |

**AC 的特殊性**：
```
Workplane / AITa：T营 100% 负责（E队 用就行）
AC：T营 提供框架 + E队 填充内容
    ├── 框架（CA 协议、ACD 结构）← T营 维护
    └── 内容（Players/Story/Times）← E队 贡献
```

> **洞见**：AC 是 A 中唯一需要 **"T营 + E队 共建"** 的部分。

---

### 洞见五：A 的终极目标是"让 A 隐形"

| 阶段 | E队 感知 | A 的状态 | UI 表现 |
|------|----------|----------|---------|
| **初期** | "网又断了" | A 可见 | 错误提示频繁 |
| **中期** | "今天不错" | A 半透明 | 偶有告警 |
| **成熟** | 专注 Mission | **A 隐形** | 无感知 |

> **洞见**：A 的成功标准不是"功能多强"，而是**"E队 多久没想起 A"**。

---

### 洞见六：A 的三支柱映射到"三种资本"

| 支柱 | 资本类型 | 折旧特征 | UI 关注点 |
|------|----------|----------|-----------|
| **Workplane** | 物理资本 | 设备/带宽成本 | 状态监控、拓扑可视化 |
| **AITa** | 金融资本 | Token 消耗 | 配额进度条、用量图表 |
| **AC** | 知识资本 | 越用越值钱 | 语境卡片、引用关系 |

> **洞见**：Workplane 和 AITa 是"消耗型"（花了就没了），AC 是"积累型"（建了就永远有）。

---

### 洞见七：A 是"桥梁"，不是"产品"

```
┌─────────┐                              ┌─────────┐
│  E队    │ ════════ A (桥梁) ═════════ │  AGI    │
│ (人类)  │                              │ (算力)  │
└─────────┘                              └─────────┘
```

| 维度 | 产品思维 | 桥梁思维（A 应采用） |
|------|----------|----------------------|
| **成功指标** | 用户爱用 | 用户不感知 |
| **价值来源** | 功能丰富 | 连接可靠 |
| **迭代重点** | 加功能 | 减故障 |

> **终极洞见**：桥的价值不在于桥本身，而在于**桥两端连接的价值**。

## §2 实施速查（前端工程师专用）

### A 分区菜单结构速查

| 子分区 | 地址段 | 入口路由 | 核心组件 |
|:------:|:------:|----------|----------|
| **Workplane** | T3.13.1 | `/admin/workplane` | Dashboard + Teamsbox |
| **AITa** | T3.13.2 | `/admin/aita` | Dashboard (含 Tab) |
| **AC** | T3.13.3 | `/admin/ac` | Dashboard + CA Registry |

### UI 状态设计检查清单

| 检查项 | Workplane | AITa | AC |
|--------|:---------:|:----:|:--:|
| 正常态 | 绿色连通图标 | 配额充足 | 语境已加载 |
| 警告态 | 延迟/抖动 | 配额 < 20% | 语境部分缺失 |
| 错误态 | 连接失败 | 配额耗尽/无权限 | 语境解析失败 |
| 操作入口 | 查看拓扑 | 查看用量 | 编辑/引用语境 |

### 故障排查 UI 流程

```
用户报障
    │
    ├─→ 检测 Workplane 状态
    │       ├─ 失败 → 显示「网络不可达」+ 联系 T营
    │       └─ 成功 ↓
    │
    ├─→ 检测 AITa 状态
    │       ├─ 配额不足 → 显示「配额告警」+ 申请入口
    │       ├─ 无权限 → 显示「权限不足」+ 联系管理员
    │       └─ 成功 ↓
    │
    ├─→ 检测 AC 状态
    │       ├─ CA 解析失败 → 显示「语境缺失」+ 跳转 AC 编辑
    │       └─ 成功 ↓
    │
    └─→ 问题在 Mission 层（非 A 分区）
```

| 路由 | T营 (super) | E队 (member) | 备注 |
|------|:-----------:|:------------:|------|

| `/admin/teamsbox` | ✅ 读写 | 👁️ 只读 | 设备管理 |
| `/admin/aita` | ✅ 读写 | 👁️ 只读 | 隐藏配额调整 |
| `/admin/ac` | ✅ 读写 | ✅ 贡献 | E队可编辑 Context |
| `/admin/ac/address` | ✅ 读写 | 👁️ 只读 | CA 注册表 |

| Tab | 功能 | E队 可见 |
|-----|------|:--------:|

### API 接口约定（参考）

| 接口 | 用途 | 返回状态 |
|------|------|----------|
| `GET /api/workplane/status` | 网络连通性检测 | `ok` / `warning` / `error` |
| `GET /api/aita/quota` | 配额剩余查询 | `{ used, total, percent }` |
| `GET /api/ac/resolve?ca=xxx` | CA 解析 | `{ acd, acc }` 或 `404` |
