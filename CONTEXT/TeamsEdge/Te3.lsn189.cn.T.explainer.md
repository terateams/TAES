## L-S-N 三层拓扑（Workplane 详解）

> **L-S-N** = Local → Server → Node = E队在地 → T营隧道 → 全球出口

| 层级 | 含义 | L2 菜单 | E队 可用的 | 确权方式 | 物理类比 |
|:----:|:-----|:--------|------------|----------|----------|
| **L** | Local | Teamsbox (T3.2.4) | CPE 边缘设备数、内置 L-S-N 路径方案 | 账单确权（专属） | 你家的路由器 |
| **S** | Server | Link Server (T3.2.5) | VPN/Proxy 服务器数、地域分布、带宽容量 | 订阅确权（共享池） | 电信的机房 |
| **N** | Node | Service Node (T3.2.6) | Apps Gateway 节点数、IP 属性（住宅/DC/移动） | 用量确权（共享池） | 海外的出口 |

**拓扑语义**：
- **L (Local)**：E队 现场的边缘设备，是请求的发起点
- **S (Server)**：T营 机房的隧道服务器，是流量的中转站
- **N (Node)**：全球分布的出口节点，是请求的终点代理

**路径矩阵**：网络能力 = L × S × N 的路径组合

**洞见**：L-S-N 不是三个独立产品，而是一条完整的数据通路。

## AGA 子能力详解（AITa 核心）

> **AGA** = Augmented GitHub Account，是 AITa 线的核心枢纽

| 子能力 | 说明 | 操作对象 |
|--------|------|----------|
| **登录账号** | TeamsCamp 的 GitHub 账号（操作主体） | 认证身份 |
| **席位采购** | Enterprise + Copilot 席位购买 | 许可证 |
| **算力池** | Premium Quota 配额管理 | Token 预算 |
| **席位分发** | 向 AO (E队 Org) 分配席位 | 授权边界 |
| **算力分发** | 向 E队 分配 $X/月 配额 | 价值契约 |

**洞见**：AGA 的本质是能力平权——

```
传统世界：能力 ∝ 资本（有钱人用好工具）
TAES 世界：能力 = 个人能力 × 托举效应

新人 × 托举 = 可以完成复杂任务
高手 × 托举 = 可以完成超复杂任务
```

## T 的深层洞见

> **洞见 1**：`T3.2.x` 是乘法关系，不是加法

```
传统 IT 采购：服务器 + 网络 + 软件 = 能力（加法，可部分使用）
TAES 托举：AITC × BAS × AGA × L × S × N = 托举效应（乘法，缺一归零）
```

这解释了为什么 T营 必须完整持有所有六项——任何一项缺失，整个托举链条断裂。

> **洞见 2**：确权方式体现资源属性

| 确权方式 | 资源属性 | 适用对象 | 计费逻辑 |
|----------|----------|----------|----------|
| **合同确权** | 固定成本 | AITC/BAS/AGA | 年费/月费 |
| **账单确权（专属）** | 资本支出 | Teamsbox | 买断/租赁 |
| **订阅确权（共享）** | 服务支出 | Link Server | 按档付费 |
| **用量确权（共享）** | 边际成本 | Service Node | 按量付费 |

> **洞见 3**：T 与 A 的分工边界

| 维度 | [T] TeamsCamp | [A] Augment |
|------|---------------|-------------|
| **问什么** | 有多少？（额度） | 怎么用？（技术） |
| **管什么** | 采购/持有/分发 | 配置/运维/监控 |
| **看什么** | 账单/合同/配额 | 状态/日志/性能 |
| **地址段** | T3.2.x | T3.3.x |

**口诀**：T 管“有没有”，A 管“通不通”。
