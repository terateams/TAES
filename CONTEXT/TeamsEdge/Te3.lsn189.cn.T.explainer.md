## TAG × BSN 双轴模型

> **核心公式**：托举效应 = TAG × BSN = T × A × G × B × S × N

| 轴 | 缩写 | 全称 | 语义 |
|:---:|:----:|------|------|
| **TAG** | T-A-G | Tenant-Azure-GitHub | 身份链（谁的资源） |
| **BSN** | B-S-N | Box-Server-Node | 路径链（怎么到达） |

### TAG：身份链（确权链）

| 位置 | 缩写 | 全称 | 说明 | 地址 |
|:----:|:----:|------|------|:------:|
| **T** | AITC | Allied Intelligent **T**enant Contract | 租户根：没有租户，无法采购 | T3.12.1 |
| **A** | ABS | **A**zure Benefit Subscription | 云平台：没有 Azure，无法付费 | T3.12.2 |
| **G** | GAA | **G**ithub Augmented Account | 代码平台：没有 GitHub，无法登录 | T3.12.3 |

**依赖链**：T → A → G（必须先有 T，才有 A，才有 G）

### BSN：路径链（路由链）

| 位置 | 缩写 | 全称 | 说明 | 地址 |
|:----:|:----:|------|------|:------:|
| **B** | Teamsbox | Teams**B**ox | 边缘盒子：本地发起点 | T3.12.4 |
| **S** | Link Server | Link **S**erver | 隧道服务器：流量中转站 | T3.12.5 |
| **N** | Service Node | Service **N**ode | 出口节点：全球代理 | T3.12.6 |

**路径链**：B → S → N（数据从 B 出发，经 S 中转，到 N 出口）

### 洞见：为什么是 TAG × BSN？

```
TAG（身份链）          BSN（路径链）
    │                      │
    ├── T: Tenant 租户根   ├── B: Box 边缘盒子
    ├── A: Azure 云平台   ├── S: Server 隧道服务
    └── G: GitHub 代码平台 └── N: Node 出口节点
          │                      │
          ↓                      ↓
      静态归属                动态路由
      “这是谁的资源”          “数据怎么流动”
```

- **TAG** 回答“这是谁的资源”——从租户到云平台到代码平台，是**确权链**
- **BSN** 回答“数据怎么流动”——从盒子到隧道到出口，是**路径链**
- **TAG × BSN** = 六因子乘法，任一为零则托举归零

## L-S-N 三层拓扑（BSN 详解）

> **BSN** = **B**ox → **S**erver → **N**ode = E队在地 → T营隧道 → 全球出口

| 层级 | TAG/BSN | L2 菜单 | E队 可用的 | 确权方式 | 物理类比 |
|:----:|:-------:|:--------|------------|----------|----------|
| **B** | Teamsbox [B] | T3.12.4 | CPE 边缘设备数、内置 B-S-N 路径方案 | 账单确权（专属） | 你家的路由器 |
| **S** | Link Server [S] | T3.12.5 | VPN/Proxy 服务器数、地域分布、带宽容量 | 订阅确权（共享池） | 电信的机房 |
| **N** | Service Node [N] | T3.12.6 | Apps Gateway 节点数、IP 属性（住宅/DC/移动） | 用量确权（共享池） | 海外的出口 |

**拓扑语义**：
- **B (Box)**：E队 现场的边缘设备，是请求的发起点
- **S (Server)**：T营 机房的隧道服务器，是流量的中转站
- **N (Node)**：全球分布的出口节点，是请求的终点代理

**路径矩阵**：网络能力 = B × S × N 的路径组合

**洞见**：BSN 不是三个独立产品，而是一条完整的数据通路。

## T 的深层洞见

> **洞见 1**：`T3.12.x` 是乘法关系，不是加法

```
传统 IT 采购：服务器 + 网络 + 软件 = 能力（加法，可部分使用）
TAES 托举：TAG × BSN = (T × A × G) × (B × S × N) = 托举效应（乘法，缺一归零）
```

这解释了为什么 T营 必须完整持有所有六项——任何一项缺失，整个托举链条断裂。

> **洞见 2**：确权方式体现资源属性

| 确权方式 | 资源属性 | 适用对象 | 计费逻辑 |
|----------|----------|----------|----------|
| **合同确权** | 固定成本 | AITC [T] / ABS [A] / GAA [G] | 年费/月费 |
| **账单确权（专属）** | 资本支出 | Teamsbox [B] | 买断/租赁 |
| **订阅确权（共享）** | 服务支出 | Link Server [S] | 按档付费 |
| **用量确权（共享）** | 边际成本 | Service Node [N] | 按量付费 |

> **洞见 3**：T 与 A 的分工边界

| 维度 | [T] TeamsCamp | [A] Augment |
|------|---------------|-------------|
| **问什么** | 有多少？（额度） | 怎么用？（技术） |
| **管什么** | 采购/持有/分发 | 配置/运维/监控 |
| **看什么** | 账单/合同/配额 | 状态/日志/性能 |
| **地址段** | T3.12.x | T3.13.x |

**口诀**：T 管“有没有”，A 管“通不通”。
