## TeamsEdge 核心价值（概念前置）

> **人操作 MU（业务对象），AI 读 AC（业务语境）**
>
> TeamsEdge 的价值 = **优化 MU 操作** + **丰富 AC 语境**

### 本体论基础：MU + AC 调和模型

> **MU 是给人操作的业务对象，AC 是给 AI 理解的业务语境。**
>
> **AC 是团队共享的心智模型，长期存在，独立于任何单个 MU。**

```
┌─────────────────────────────────────────────────────────────────┐
│  MU + AC 调和模型                                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  两个独立存在的实体，通过 CA 关联：                             │
│                                                                 │
│  ┌─────────────────────┐      CA      ┌─────────────────────┐  │
│  │  MU (业务对象)      │ ←─────────→ │  AC (团队心智)      │  │
│  ├─────────────────────┤              ├─────────────────────┤  │
│  │ • 持久存在          │              │ • 持久存在          │  │
│  │ • 给人操作          │              │ • 给 AI 理解        │  │
│  │ • 存储在数据库      │              │ • 存储在 CONTEXT/   │  │
│  │ • 包含：属性/状态   │              │ • 包含：意义/背景   │  │
│  └─────────────────────┘              └─────────────────────┘  │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  MU 实例：Player, E队, AO, AR, Teamsbox, Mission...            │
│  AC 特性：独立于 MU，与团队同寿，多个 MU 可共享同一 AC          │
│  CA 作用：关联 MU 与其对应的 AC                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### MU (Mission Unit) — 业务对象

```
┌─────────────────────────────────────────────────────────────────┐
│  MU (Mission Unit) — 可操作的业务对象                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  定义：Mission 中可操作的业务实体                               │
│                                                                 │
│  MU 实例（与 TAES 菜单对齐）：                                  │
│  ├── [E] Player, E队, AO, AR, Publisher                         │
│  ├── [T] AITC, BAS, AGA, Teamsbox, Link Server                  │
│  ├── [A] Workplane配置, AITa配额, AC实例                        │
│  └── [S] Mission, Artifact, Intelligence                        │
│                                                                 │
│  属性：                                                         │
│  ├── mu_id: string        // 业务对象标识                       │
│  ├── mu_type: enum        // 对象类型（player, team, org...）   │
│  ├── ca: ContextAddress   // 关联的 AC 地址                     │
│  └── payload: object      // 业务数据                           │
│                                                                 │
│  双重表达：                                                     │
│  • UI → 给人操作（CRUD）                                        │
│  • AC → 给 AI 理解（业务语境）                                  │
│                                                                 │
│  用户体验：                                                     │
│  • 用户通过 UI 操作 MU                                          │
│  • AI 通过 AC 理解 MU 的业务含义                                │
│  • CA 自动关联 MU 与 AC                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### CA (Context Address) — 语境地址

> **CA 之于 Context，如同 IP 之于 Host**

```
Context Protocol Specification v1.0

URI Scheme: ca://

Format: ca://{namespace}/{type}/{id}

Components:
  - namespace: AC 所属的命名空间（org, repo, system）
  - type:      AC 类型（player, mission, framework, etc.）
  - id:        AC 实例标识

Examples:
  ca://suibe-org/player/zhang      → 队员张三的 Context
  ca://mar-001/mission/outline     → MAR-001 的 Mission 大纲
  ca://taes/framework/clear        → TAES 的 CLEAR 框架
  ca://system/template/report      → 系统级报告模板

Resolution:
  CA → AC Resolver → ACD (Data) → ACC (Card)
```

```
┌─────────────────────────────────────────────────────────────────┐
│  MU + AC 架构（调和版）                                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  MU (业务对象)                 AC (团队心智)                     │
│  ══════════════                ══════════════                   │
│  存储在数据库/GitHub           存储在 CONTEXT/ 目录             │
│  通过 UI 给人操作              通过 CA 给 AI 读取               │
│  包含：属性、状态、关系        包含：意义、背景、约束           │
│  生命周期：持久                生命周期：与团队同寿             │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│      MU₁        MU₂        MU₃        MU₄                      │
│       │          │          │          │                        │
│       └──────────┴────┬─────┴──────────┘                        │
│                       │ CA                                      │
│                       ↓                                         │
│              ┌─────────────────┐                                │
│              │       AC        │                                │
│              │  (团队共享心智) │                                │
│              │   长期存在      │                                │
│              └─────────────────┘                                │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  Player "张三"    ←── ca://suibe-org/player/zhang ──→           │
│                        张三的画像、技能、偏好                   │
│                                                                 │
│  E队 "MAR-001"    ←── ca://mar-001/team/info ──→                │
│                        团队背景、Mission、方法论                │
│                                                                 │
│  ─────────────────────────────────────────────────────────────  │
│                                                                 │
│  人机协作 = 人操作 MU + AI 读 AC + AI 辅助人                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| 术语 | 全称 | 定义 | 给谁用 | 生命周期 |
|:----:|------|------|:------:|:--------:|
| **MU** | Mission Unit | 可操作的业务对象 | 人类 | 持久 |
| **AC** | Augment Context | 团队心智（MU 的语境层） | AI | 与团队同寿 |
| **CA** | Context Address | MU 与 AC 的关联指针 | 系统 | — |
| **CP** | Context Protocol | 寻址协议规范 | — | — |
