# TeamsEdge 用户角色与使用场景

> 本文档定义 TeamsEdge 的目标用户、权限模型与核心使用场景。

---

## 1. 系统定位

> **TeamsEdge 是 T营内部运营系统**，T营人员是主要操作者，E队 Owner 仅作校对/审批/有限申请之用。

```
┌─────────────────────────────────────────────────────────────┐
│                      TeamsEdge 系统                          │
├─────────────────────────────────────────────────────────────┤
│  主要操作者：T营（管理层 / 业务员 / 工程师 / 财法）            │
│  有限访问者：E队 Owner（只读 + 有限写入）                      │
│  不接触系统：E队 Member（直接使用 Copilot/GitHub/Teams）       │
└─────────────────────────────────────────────────────────────┘
```

## 2. 用户角色定义

### 2.1 角色总览

| 角色 | 归属 | 典型身份 | 权限级别 | 核心关注菜单 |
| :--- | :--- | :--- | :--- | :--- |
| **T营 管理层** | T营 | 总监、区域负责人 | 审批+监控 | Dashboard、全局 KPI、审批流 |
| **T营 业务员** | T营 | 销售、客户成功经理 | 读写 | EdgeTeams、Mission、TeamsCamp > Entities |
| **T营 工程师** | T营 | 运维、技术支撑 | 读写 | Infrastructure、System、AI Intelligence |
| **T营 财法** | T营 | 财务、法务专员 | 读写 | Credits & Billing、TeamsCamp > Entities |
| **E队 Owner** | E队 | 客户团队负责人 | **只读+有限写入** | EdgeTeams（本队）、Mission（本队）、Foundry |

### 2.2 T营 管理层权限说明

> **职责**：全局监控、审批决策、绩效管理

| 权限类型 | 具体操作 | 说明 |
| :--- | :--- | :--- |
| **监控** | 全局 Dashboard | E队 状态、Mission 进度、算力消耗、营收 |
| **审批** | Mission 申请审批 | E队 Owner 发起 → 管理层审批 → 业务员执行 |
| **审批** | 大额资源申请 | 超阈值的算力/席位申请需管理层批准 |
| **KPI** | 团队绩效看板 | 业务员/工程师的服务指标 |

### 2.3 T营 业务员权限说明

> **职责**：E队 全生命周期管理、Mission 执行

| 权限类型 | 具体操作 | 说明 |
| :--- | :--- | :--- |
| **创建** | 新建 E队 | 初始化 Org/Repo/Security Group |
| **创建** | 新建 Mission | 生成 Mission Repo，配置 MISSION.md |
| **管理** | E队 配置 | 成员、角色、权限调整 |
| **管理** | 合同主体 | TeamsCamp > Entities 维护 |
| **执行** | 审批后执行 | 管理层批准后执行资源分配 |

### 2.4 T营 工程师权限说明

> **职责**：基础设施运维、AI 算力管理

| 权限类型 | 具体操作 | 说明 |
| :--- | :--- | :--- |
| **配置** | 网络托举 | Infrastructure > Workplane |
| **配置** | 域名管理 | System > Domain |
| **监控** | AI 用量 | AI Intelligence > Usage / Eval |
| **运维** | 系统维护 | System > Backup / Setting |
| **响应** | 工单处理 | Notification > Ticket |

### 2.5 T营 财法权限说明

> **职责**：财务结算、合规管理

| 权限类型 | 具体操作 | 说明 |
| :--- | :--- | :--- |
| **管理** | 发票处理 | Credits & Billing > Invoice |
| **管理** | 合同主体 | TeamsCamp > Entities |
| **查看** | 账单明细 | Credits & Billing 全局视图 |
| **审核** | 合规检查 | 合同、发票合规性审核 |

### 2.6 E队 Owner 权限说明

> **职责**：本队配置校对、交付物确认、账单核对

| 权限类型 | 具体操作 | 说明 |
| :--- | :--- | :--- |
| **只读** | 查看本队配置 | EdgeTeams > 本队信息 |
| **只读** | 查看交付物 | Foundry > Artifacts（下载 PDF/App 包） |
| **只读** | 查看账单 | Credits & Billing > 本队消费明细 |
| **有限写入** | 申请新 Mission | 提交申请 → T营 业务员/管理层审批 |
| **有限写入** | 提交反馈/工单 | Notification > Ticket → T营 工程师处理 |
| **有限写入** | 确认交付物 | 签收确认（Evidence 留痕） |

## 5. 设计决策说明

| 设计决策 | 理由 |
| :--- | :--- |
| **T营四分（管理/业务/技术/财法）** | 增加管理层，支撑审批流与全局监控；职能分离，权责清晰 |
| **E队 Owner 有限写入** | "申请-审批"模式，E队可发起，T营审批执行，符合服务流程 |
| **E队 Owner 有限区域** | 仅可见本队数据，用于配置校对、交付物确认、账单核对 |
| **无 E队 Member** | E队 成员直接使用 Copilot/GitHub/Teams，不接触 TeamsEdge |
| **审批流集中** | 所有审批统一在 Notification > Approval，便于管理层处理 |

## 6. 审批流设计

### 6.1 Mission 申请审批流

```
E队 Owner 发起申请
       ↓
T营 业务员 初审（可选）
       ↓
T营 管理层 审批
       ↓
   ┌───┴───┐
   ↓       ↓
 批准     驳回
   ↓       ↓
业务员执行  通知Owner
```

### 6.2 大额资源申请审批流

```
T营 业务员/工程师 发起申请
       ↓
T营 管理层 审批
       ↓
   ┌───┴───┐
   ↓       ↓
 批准     驳回
   ↓       ↓
自动执行   通知申请人
```
