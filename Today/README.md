## 📂 三体结构

```
/Today/               ← 根目录级别
├── in.md             ← 进（收集一切，日清后清空）
├── out.md            ← 出（处理结果，分发后清空）
└── log.md            ← 留（追加归档，永不清空）
```

### 各文件职责

| 文件 | 中文 | Unix 类比 | 生命周期 | 用途 |
|------|------|-----------|----------|------|
| **in.md** | 进 | stdin | 日清 | 收件箱：想法、任务、剪贴 |
| **out.md** | 出 | stdout | 日清 | 发件箱：处理结果、待分发 |
| **log.md** | 留 | stderr | 永久 | 审计轨迹：归档、追溯 |

---

## 🔄 日清工作流

```
外部世界 → in.md（收集）→ AI 处理 → out.md（产出）→ 分发到目标
                              ↓
                          log.md（归档）
```

1. **进 (in)**：所有输入先进 in.md，不分类、不判断
2. **出 (out)**：处理结果进 out.md，等待分发到最终位置
3. **留 (log)**：日清时，有价值的追加到 log.md，然后清空 in/out

---

## 🎯 设计原则

| 原则 | 实现 |
|------|------|
| **零摩擦** | 固定位置，无需寻找 |
| **日清制** | in/out 每日清空，log 永久追加 |
| **认知友好** | 三个文件 = 人类工作记忆极限 |
| **Unix 哲学** | 输入/输出/日志 = stdin/stdout/stderr |
| **设备无关** | 任何设备上 /Today 都是同一个位置 |
