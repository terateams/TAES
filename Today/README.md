# Today — 你的数字工作台

> **每天从这里开始，每天在这里清空。**

---

## 这是什么？

Today 是你的**数字收发室**——所有工作先进这里，处理完再发出去。

就像办公室前台：
- 📥 收到的文件先放收件篮
- 📤 处理完的文件放发件篮
- 📋 重要的事情登记在册

---

## 三个文件，三个动作

| 文件 | 动作 | 什么时候用 |
|------|------|------------|
| **in.md** | 收 | 有想法？先写这里。收到任务？先贴这里。 |
| **out.md** | 发 | 处理完了？结果放这里，等着发到该去的地方。 |
| **log.md** | 存 | 每天结束，重要的追加到这里，永久保存。 |

---

## 每天怎么用？

### 早上：打开 in.md

- 昨天没处理完的任务在这里
- 新想到的事情直接写进来
- 收到的邮件、消息，关键内容贴进来

### 白天：边工作边用

```
收到新任务 → 贴进 in.md
完成一项工作 → 结果放 out.md
```

### 下班前：清空

1. **in.md 里的任务**
   - 做完了？删掉
   - 没做完？留着明天继续

2. **out.md 里的结果**
   - 发给该收的人（邮件、群、系统）
   - 发完就删

3. **重要的事情**
   - 追加到 log.md 底部，记录日期
   - 这样以后能找到

---

## 举个例子

### in.md（早上 9:00）

```markdown
# in

> 进 — 收集一切，日清后清空

---

## 待办
- 给张总回复合同问题
- 下午 3 点开会，准备材料
- 报销单还没贴

## 刚收到
- 李姐发来的客户名单（见附件）
- IT 说系统今晚维护
```

### out.md（下午 5:00）

```markdown
# out

> 出 — 处理结果，分发后清空

---

## 待发送
- [x] 合同回复 → 已发张总邮箱
- [ ] 会议纪要 → 待发群里
- [ ] 报销单照片 → 待上传系统
```

### log.md（追加记录）

```markdown
# log

> 留 — 追加归档，永不清空

---

## 2026-01-22
- 张总合同确认，约下周签约
- 报销 ¥328，单号 #20260122-01

## 2026-01-21
- 客户 A 项目启动会完成
- 新同事小王入职
```

## 记住这三句话

1. **所有东西先进 in.md** — 别想着分类，先收进来
2. **做完的结果放 out.md** — 发出去之前放这里
3. **每天清空，重要的存 log.md** — 明天又是新的一天

## 常见问题

**Q：我习惯用便签/笔记本，为什么要用这个？**

A：便签会丢，笔记本会满。Today 永远在这里，而且 AI 能帮你处理。

**Q：log.md 会不会越来越长？**

A：会。但这正是它的价值——你的工作轨迹全在这里，随时可查。

**Q：如果一件事几天都没做完怎么办？**

A：留在 in.md 就好。它会一直提醒你，直到你做完或者决定不做。
