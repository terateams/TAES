# Today — 第二大脑的前台

> **Today 是协议，不是实现。**
>
> 三个文件 = stdin / stdout / stderr
> 进、出、留 — 认知科学的黄金数字

## 📂 三体结构

```
Today/
├── in.md     ← 进（收集一切，日清后清空）
├── out.md    ← 出（处理结果，分发后清空）
└── log.md    ← 留（追加归档，永不清空）
```

### 各文件职责

| 文件 | 中文 | Unix 类比 | 生命周期 | 用途 |
|------|------|-----------|----------|------|
| **in.md** | 进 | stdin | 日清 | 收件箱：想法、任务、剪贴 |
| **out.md** | 出 | stdout | 日清 | 发件箱：处理结果、待分发 |
| **log.md** | 留 | stderr | 永久 | 审计轨迹：归档、追溯 |

---

## 🔄 日清工作流

```
外部世界 → in.md（收集）→ AI 处理 → out.md（产出）→ 分发到目标
                              ↓
                          log.md（归档）
```

1. **进 (in)**：所有输入先进 in.md，不分类、不判断
2. **出 (out)**：处理结果进 out.md，等待分发到最终位置
3. **留 (log)**：日清时，有价值的追加到 log.md，然后清空 in/out

---

## 🎯 设计原则

| 原则 | 实现 |
|------|------|
| **零摩擦** | 固定位置，无需寻找 |
| **日清制** | in/out 每日清空，log 永久追加 |
| **认知友好** | 三个文件 = 人类工作记忆极限 |
| **Unix 哲学** | 输入/输出/日志 = stdin/stdout/stderr |
| **设备无关** | 任何设备上 /Today 都是同一个位置 |

## 📍 架构位置

```
/Today/                  ← 根目录级别，不依赖任何 Repo
├── in.md                ← stdin：所有输入的入口
├── out.md               ← stdout：所有输出的出口
└── log.md               ← stderr：所有轨迹的归档
```

**与 Repo 的关系**：

| 概念 | 位置 | 性质 |
|------|------|------|
| **个人 Today** | `/Today/` | 系统基础设施，永远存在 |
| **项目 Today** | `{Repo}/Today.md` | 项目日志，可选配置 |

## 日清规则

> **Today 的生命周期是 24 小时。**

每日结束时：

| 内容类型 | 处理方式 |
|----------|----------|
| **有价值** | 归档到对应 Repo 或 `Archive/` |
| **可丢弃** | 直接删除 |
| **待续** | 保留在 Today.md，标注日期 |

## 在 MAR 架构中的位置

```
MISSION.md      →  "我要做什么"（战略意图 — 年度/季度级）
    ↓
Today/Today.md  →  "我今天在做什么"（战术意图 — 当日级）
    ↓
Teams/Artifacts →  "我做出了什么"（交付物）
```

**Today 填补了意图与交付之间的断层。**

---

## 在 E=MC²AI 中的位置

```
E = MC²AI

M (Mission)     →  MISSION.md（为什么做）
C¹ (Connection) →  Today/（此刻在做）  ← Today 属于 Connection 层
C² (Context)    →  CONTEXT/（背景是什么）
AI (Allied AI)  →  .github/copilot-instructions.md
E (Empower)     →  Teams/Artifacts/（做出了什么）
```

**Today 属于 Connection 层**：它连接的是**当下的人**与**当下的意图**——这是最短、最热、最频繁的连接。
