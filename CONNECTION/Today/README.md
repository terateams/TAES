# Today — Repo 的工位

> **README 让 Repo 被理解，Today 让 Repo 被使用。**
>
> 没有 README 的 Repo 是"无名"的；没有 Today 的 Repo 是"无人"的。

---

## 本质定位

```
┌─────────────────────────────────────────────────────────────────┐
│  Today 的本质                                                    │
├─────────────────────────────────────────────────────────────────┤
│  不是"文档"，是"工作台面"                                       │
│  不是"归档"，是"当日缓冲"                                       │
│  不是"交付物"，是"意图的草稿纸"                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## README 与 Today 的对称性

| 维度 | README | Today |
|------|--------|-------|
| **时间属性** | 永恒（Repo 生命周期） | 当日（24小时周期） |
| **内容属性** | 声明式（是什么） | 过程式（在做什么） |
| **读者** | 外部人（第一次来） | 自己（当下工作） |
| **AI 角色** | 理解 Repo 的入口 | 执行任务的上下文 |
| **核心问题** | 这个 Repo **是**什么？ | 这个 Repo **今天在做**什么？ |

```
README  →  Repo 的"名片"（静态的身份宣言）
Today   →  Repo 的"工位"（动态的意图现场）
```

---

## 为什么 Today 是标准配置？

### 范式转移：Repo 从"代码仓库"变成"工作空间"

| 传统 Repo | AI 时代 Repo |
|-----------|--------------|
| 存储代码 | **承载工作** |
| 版本控制 | **意图追溯** |
| 协作开发 | **人机协同** |
| README 足够 | **README + Today 缺一不可** |

**深层原因**：AI 改变了"工作"的定义——工作不再是"写代码"，而是"与 AI 对话产出价值"。Today 是这种对话的**物理载体**。

### Today 解决的核心痛点

| 痛点 | 没有 Today | 有 Today |
|------|------------|----------|
| **上下文丢失** | 每次对话重新解释 | 当日上下文持久化 |
| **Copy/Paste 分散** | 剪贴板、笔记本、聊天记录 | 统一入口 |
| **意图不可追溯** | "我昨天在干嘛？" | 归档可查 |
| **AI 理解成本** | 每次重建理解 | Today 就是上下文 |

## 目录结构

```
/Today/
├── README.md       # 本文件：Today 的定位与使用说明
├── Today.md        # 主工作台：当日工作内容
├── Clipboard/      # 临时片段：跨 Repo 引用的代码/文本
├── Scratch/        # 草稿区：未成型的想法
└── Archive/        # 日清归档：有价值内容的历史存档
```

| 文件/目录 | 生命周期 | 用途 |
|-----------|:--------:|------|
| `Today.md` | 24 小时 | 当日工作的"草稿纸"，AI 对话片段、临时笔记 |
| `Clipboard/` | 临时 | 需要跨 Repo 引用的代码/文本片段 |
| `Scratch/` | 临时 | 未成型的想法、实验性内容 |
| `Archive/` | 永久 | 有价值的历史内容（日清后归档） |

---

## 日清规则

> **Today 的生命周期是 24 小时。**

每日结束时：

| 内容类型 | 处理方式 |
|----------|----------|
| **有价值** | 归档到对应 Repo 或 `Archive/` |
| **可丢弃** | 直接删除 |
| **待续** | 保留在 Today.md，标注日期 |

## 在 MAR 架构中的位置

```
MISSION.md      →  "我要做什么"（战略意图 — 年度/季度级）
    ↓
Today/Today.md  →  "我今天在做什么"（战术意图 — 当日级）
    ↓
Teams/Artifacts →  "我做出了什么"（交付物）
```

**Today 填补了意图与交付之间的断层。**

---

## 在 E=MC²AI 中的位置

```
E = MC²AI

M (Mission)     →  MISSION.md（为什么做）
C¹ (Connection) →  Today/（此刻在做）  ← Today 属于 Connection 层
C² (Context)    →  CONTEXT/（背景是什么）
AI (Allied AI)  →  .github/copilot-instructions.md
E (Empower)     →  Teams/Artifacts/（做出了什么）
```

**Today 属于 Connection 层**：它连接的是**当下的人**与**当下的意图**——这是最短、最热、最频繁的连接。

## 使用示例

```markdown
## 当前任务
- [ ] 完成 Te3 菜单 TAES 对齐
- [ ] 优化 Allied Tokyo 银行方案

## AI 对话片段
（从 Copilot 对话中 Copy 的有价值内容）

## 待处理
- 明天跟进 XXX
```

### 日清归档示例

```
Today.md (2026-01-22 内容)
    ↓
Archive/2026-01-22.md（归档）
    ↓
Today.md（清空，开始新的一天）
```
