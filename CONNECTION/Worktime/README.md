# ⏰ Worktime - 工作时间层

> CONNECTION > Worktime
> 
> **5W: When — 什么时候需要连接？**
> 
> *管理的稀缺性：时间稀缺（并非所有时段都需要全量连接）*

## 概述

Worktime 定义了 E队的工作时段和 Connection 资源调度策略。

**核心理念：按需连接，弹性成本。**

> 传统网络假设 24/7 永续在线。Worktime 创造了"时间感知型网络架构"——连接是有生命周期的，会呼吸、会休眠、会醒来。

## 状态定义

```
┌─────────────────────────────────────────────────────────┐
│                  CONNECTION 状态机                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────┐    上班    ┌─────────┐    下班    ┌───────┐│
│  │ STANDBY │ ─────────→ │ ACTIVE  │ ─────────→ │STANDBY││
│  │  待机   │ ←───────── │  工作中  │ ←───────── │ 待机  ││
│  └─────────┘   紧急召回  └─────────┘   临时加班  └───────┘│
│       │                      │                     │    │
│       │       长期离线       │                     │    │
│       └──────────────────────┼─────────────────────┘    │
│                              ↓                          │
│                        ┌─────────┐                      │
│                        │ OFFLINE │                      │
│                        │  离线   │                      │
│                        └─────────┘                      │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

| 状态 | 说明 | 资源消耗 |
|------|------|----------|
| **ACTIVE** | 工作中，全量 Connection | ¥¥¥ |
| **STANDBY** | 待机，基本 Connection | ¥ |
| **OFFLINE** | 离线，无 Connection | ¥0 |

## 成本模型

```
传统模式：24h × 固定带宽 × 人数 = 高固定成本
MR 模式：Σ(ACTIVE时长 × 费率) + Σ(STANDBY时长 × 低费率) = 弹性成本

节省 = (传统成本 - MR成本) / 传统成本 × 100%
典型节省：40-60%
```
