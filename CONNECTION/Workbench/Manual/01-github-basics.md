# GitHub 基础

> 预计学习时间：10 分钟

## 什么是 GitHub？

GitHub 是一个**代码托管和协作平台**，但不仅仅用于代码——我们用它来管理任务、文档和团队协作。

## 核心概念

### 1. 仓库（Repository）

仓库就像一个**项目文件夹**，包含所有相关文件和历史记录。

```
仓库 = 项目的所有文件 + 修改历史 + 协作工具
```

### 2. 提交（Commit）

每次保存修改就是一次**提交**，就像游戏存档一样。

```
提交 = 一次修改的快照 + 说明信息
```

### 3. 分支（Branch）

分支是**独立的工作空间**，可以并行开发而不互相干扰。

```
main 分支 ─────────────────────→  （主线，稳定版本）
              │
              └── feature 分支 ──→  （开发新功能）
```

### 4. Issue（议题）

Issue 用于**记录任务、Bug 或讨论**。

```
Issue = 任务卡片 + 讨论区 + 状态追踪
```

### 5. Pull Request（PR）

PR 是**合并代码的请求**，包含代码审查和讨论。

```
PR = "我完成了这些修改，请审查后合并"
```

## 基本操作

### 在网页上操作（推荐新手）

1. **查看文件**：直接点击文件名
2. **编辑文件**：点击铅笔图标 ✏️
3. **创建 Issue**：点击 "Issues" → "New issue"
4. **评论讨论**：在 Issue 或 PR 底部输入评论

### 使用 VS Code（推荐）

1. 安装 VS Code
2. 安装 GitHub 扩展
3. 登录 GitHub 账号
4. 直接在 VS Code 中编辑和提交
