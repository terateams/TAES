# 团队协作流程

> 预计学习时间：10 分钟

## 协作原则

```
📌 一切皆可追溯
📌 沟通留痕
📌 小步快跑
```

## 日常协作流程

### 1️⃣ 领取任务

```
查看 Issues → 找到分配给你的任务 → 开始工作
```

**操作步骤**：
1. 点击 "Issues" 标签
2. 筛选 `assignee:你的用户名`
3. 点击任务查看详情

### 2️⃣ 更新进度

在 Issue 中留下评论，让团队知道你的进展：

```markdown
## 进度更新 2025-12-07

✅ 已完成：
- 完成了 XXX

🔄 进行中：
- 正在做 YYY

⏳ 计划：
- 明天完成 ZZZ

❓ 问题/阻塞：
- 需要 @某人 帮忙确认 XXX
```

### 3️⃣ 提交成果

**方式一：直接编辑（简单修改）**
1. 在 GitHub 网页上编辑文件
2. 填写提交信息
3. 点击 "Commit changes"

**方式二：Pull Request（重要修改）**
1. 创建新分支
2. 在分支上修改
3. 创建 PR 请求审查
4. 审查通过后合并

### 4️⃣ 完成任务

1. 在 Issue 中留言说明已完成
2. 关闭 Issue（或等负责人关闭）

## 沟通规范

### @提及

使用 `@用户名` 通知特定人员：

```markdown
@zhangsan 请帮忙审查一下这个方案
```

### 标签使用

| 标签 | 含义 |
|------|------|
| `task` | 任务 |
| `bug` | 问题/缺陷 |
| `question` | 需要讨论 |
| `urgent` | 紧急 |
| `help-wanted` | 需要帮助 |

### Emoji 快速反馈

| Emoji | 含义 |
|-------|------|
| 👍 | 同意/已阅 |
| 🎉 | 祝贺/完成 |
| ❓ | 有疑问 |
| 👀 | 正在看 |

## 常见场景

### 场景：发现问题

1. 创建新 Issue
2. 描述问题和复现步骤
3. 添加 `bug` 标签

### 场景：有个想法

1. 在 Discussions 中发起讨论
2. 或创建 Issue 并添加 `question` 标签

### 场景：需要帮助

1. 在 Issue 中 @相关人员
2. 或使用 Copilot Chat 提问
