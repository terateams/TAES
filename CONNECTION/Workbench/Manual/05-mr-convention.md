# MAR 规范（Mission Augmented Repo）

> 预计学习时间：5 分钟

## 什么是 MAR？

> **MAR = Mission（使命）+ Augmented（增强）+ Repo（仓库）**
>
> **TAES**：協力營托举E队——AI真干活
>
> **MAR**：AI 托举执行，Mission 由你驾驭
>
> 这不是未来。这是 2026。

**核心洞见**：没有 AI (Copilot) 托举，普通商业工作者无法操作 VSCode，也无法从 Repo 中受益。**Augmented（增强）** 不是可选功能，而是**必要前提**。

我们使用 GitHub 仓库 + AI 来管理公司的任务和项目。

## 仓库结构

```
MAR/
├── MISSION.md             # [M] 任务宣言
├── README.md              # 架构说明书
├── MISSION/               # [M] 执行内核
│   ├── docs/              #     文档
│   └── assets/            #     资源
├── CONNECTION/            # [C¹] 连结平面
├── CONTEXT/               # [C²] 语境容器
├── AI/                    # [AI] 智能中枢（MAR的核心增强层）
└── Teams/                 # [E] EdgeTeams 交付
```

## 必读文件

| 文件 | 说明 | 何时阅读 |
|------|------|----------|
| **MISSION.md** | 任务核心信息 | 加入任务时第一时间阅读 |

## 工作规范

### 1. 任务领取

1. 查看 MISSION.md 了解任务背景
2. 查看 Issues 找到你的任务
3. 在 Issue 中回复确认

### 2. 进度同步

- **每日**：在 Issue 中更新进度（如有阻塞）
- **完成时**：提交成果并留言

### 3. 文件规范

| 内容 | 存放位置 |
|------|----------|
| 文档 | `MISSION/docs/` |
| 图片/资源 | `MISSION/assets/` |
| 最终交付物 | `Teams/` |

### 4. 命名规范

- 文件名：小写字母，用 `-` 连接
- 示例：`meeting-notes-2025-12-07.md`

### 5. 提交信息

使用有意义的提交信息：

```
✅ 好的提交信息：
- 添加需求文档 v1.0
- 修复登录页面样式问题
- 更新 API 接口文档

❌ 不好的提交信息：
- update
- fix
- 111
```

**推荐使用 emoji 前缀**：

| Emoji | 含义 |
|-------|------|
| 📝 | 文档更新 |
| ✨ | 新功能 |
| 🐛 | 修复 Bug |
| 🎨 | 样式/格式 |
| 🔧 | 配置修改 |

## 协作礼仪

1. **及时响应**：被 @ 后尽快回复
2. **主动同步**：遇到问题及时沟通
3. **尊重他人**：友善讨论，对事不对人
4. **留下记录**：重要决定在 Issue/PR 中记录

## 遇到问题？

1. 查看 `learn/` 目录的学习资料
2. 在 Discussions 中提问
3. 使用 Copilot Chat（`请以同学的身份帮我...`）
4. 联系任务负责人
