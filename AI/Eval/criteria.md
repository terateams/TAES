# 🎯 AITa 评估标准

> AITa 准入与评估 — 我们的核心资产

---

## 定义

**AITa 评估**：一条 AITa 路径需要满足什么条件，才能交付给 E 队？

```
AITa = CONNECTION 依赖 + 红利源 + AI 能力
```

---

## 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| **CONNECTION 就绪** | 35% | 账号、网络、付费通道是否可用 |
| **红利稳定性** | 30% | 红利源是否持续、额度是否充足 |
| **AI 能力匹配** | 20% | 模型能力是否满足业务场景 |
| **运维成本** | 15% | 维护该 AITa 的人力投入 |

## 准入标准

| 条件 | 要求 |
|------|------|
| CONNECTION 依赖 | 全部满足 |
| 红利有效期 | ≥ 3 个月 |
| 红利覆盖率 | ≥ 80% E 队需求 |
| AI 能力评分 | ≥ ⭐⭐⭐ |

---

## 评估流程

```
新 AITa 识别
    │
    ▼
┌─────────────────┐
│ CONNECTION 检查 │  ← 依赖是否就绪？
└────────┬────────┘
         │
    ┌────┴────┐
    │ 不满足  │ 满足
    ▼         ▼
  暂缓    ┌─────────────┐
          │  红利评估   │  ← 红利是否稳定？
          └──────┬──────┘
                 │
            ┌────┴────┐
            │ 不稳定  │ 稳定
            ▼         ▼
          暂缓    ┌─────────────┐
                  │  能力测试   │
                  └──────┬──────┘
                         │
                    进入 AITa 清单
```
