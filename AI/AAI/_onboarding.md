# 🚀 新大厂接入流程

> 当新的 AI 大厂提供红利机会时

---

## 触发条件

当以下情况发生时，启动接入流程：

- 新大厂提供免费额度/红利
- 现有大厂推出新产品线
- E队有新的 AI 需求

---

## 接入流程

### Phase 1: 评估 (1-2周)

```
发现机会
    ↓
初步评估
├── 红利来源是什么？
├── 获取复杂度？
├── 与现有 AITa 是否互补？
└── CONNECTION 依赖是什么？
    ↓
决策：是否接入
```

**评估清单：**
- [ ] 红利来源明确（IUR/Benefit/代付/其他）
- [ ] 获取成本可接受
- [ ] 有明确使用场景
- [ ] CONNECTION 依赖可满足
- [ ] 不与现有 AITa 严重重叠

→ 参考 [Eval/criteria.md](../Eval/criteria.md)

### Phase 2: 准备 (1-2周)

```
决定接入
    ↓
准备工作
├── 建立合作关系/账号
├── 确认红利获取流程
├── 准备 CONNECTION 依赖
└── 创建文档框架
    ↓
试点测试
```

**准备清单：**
- [ ] 合作关系/账号就绪
- [ ] 红利获取流程文档化
- [ ] CONNECTION 依赖确认就绪
- [ ] 创建 AITa/[大厂].md（使用 _template.md）
- [ ] 创建 AAI/[大厂]/ 目录
- [ ] 创建 _e2e/onboard_[产品].md

### Phase 3: 试点 (2-4周)

```
试点开始
    ↓
小范围使用
├── 选择 5-10 人试点
├── 收集反馈
├── 完善文档
└── 验证红利流程
    ↓
试点总结
```

**试点清单：**
- [ ] 试点用户名单确定
- [ ] 席位/账号分配
- [ ] 使用培训
- [ ] 反馈收集机制
- [ ] 问题记录与解决

### Phase 4: 推广 (持续)

```
试点成功
    ↓
正式推广
├── 更新 AITa/README.md 清单
├── 更新 AI/README.md
├── 内部公告
└── 开放申请
    ↓
持续运维
```

**推广清单：**
- [ ] 所有文档完成
- [ ] AITa 清单更新
- [ ] 内部公告发布
- [ ] 申请流程就绪
- [ ] AAI 运维流程就绪

## 文档创建清单

新大厂接入需创建：

| 文档 | 位置 | 用途 |
|------|------|------|
| [大厂].md | AITa/ | 使用说明 |
| _models.md | AITa/ | 更新模型信息 |
| README.md | AAI/[大厂]/ | 运维概览 |
| [红利源].md | AAI/[大厂]/ | 具体运维流程 |
| onboard_[产品].md | _e2e/ | 用户上手指南 |
| troubleshoot.md | _e2e/ | 更新问题排查 |

## 示例：接入 Anthropic Claude

假设 Anthropic 提供企业合作红利：

```
Phase 1: 评估
├── 红利源: Anthropic Partner Program
├── 复杂度: ⭐⭐⭐⭐
├── 场景: 深度推理、长文本
└── CONNECTION: Workpass/anthropic + Workplane

Phase 2: 准备
├── AITa/Anthropic.md
├── AAI/Anthropic/partner.md
└── _e2e/onboard_claude.md

Phase 3: 试点
├── 10 人试点 2 周
└── 收集反馈

Phase 4: 推广
├── 更新清单（3 → 4 个 AITa）
└── 开放申请
```
