# 🛡️ 数据安全

> 企业数据与 AI 的边界

---

## 数据分类

| 级别 | 定义 | 示例 |
|------|------|------|
| **L1 公开** | 可公开的信息 | 公司介绍、产品文档 |
| **L2 内部** | 仅限内部使用 | 内部流程、一般工作文档 |
| **L3 机密** | 敏感业务信息 | 财务数据、战略规划 |
| **L4 绝密** | 最高保密级别 | 核心商业机密、客户隐私 |

---

## AI 使用策略

| 数据级别 | GitHub Copilot | M365 Copilot | Google Gemini |
|----------|----------------|--------------|---------------|
| **L1 公开** | ✅ 可用 | ✅ 可用 | ✅ 可用 |
| **L2 内部** | ✅ 可用 | ✅ 可用 | ⚠️ 谨慎 |
| **L3 机密** | ⚠️ 谨慎 | ⚠️ 谨慎 | ❌ 禁止 |
| **L4 绝密** | ❌ 禁止 | ❌ 禁止 | ❌ 禁止 |

**说明：**
- ✅ 可用：正常使用
- ⚠️ 谨慎：需脱敏或经审批
- ❌ 禁止：不得输入 AI

## 数据流向

### GitHub Copilot

```
代码 → GitHub Copilot → 云端模型 → 建议
         ↑
    企业版：不用于训练
```

**安全保障：**
- Copilot Business/Enterprise 承诺不用于模型训练
- 代码在传输中加密
- 建议仅基于当前上下文

### M365 Copilot

```
文档 → M365 Copilot → Azure OpenAI → 结果
              ↑
    数据留在 M365 边界内
```

**安全保障：**
- 数据在 Microsoft 365 合规边界内
- 遵守企业 DLP 策略
- 不与其他租户共享

### Google Gemini

```
输入 → Gemini → Google 云端 → 结果
          ↑
    代付账号：检查隐私设置
```

**安全保障：**
- 确认隐私设置正确
- 避免输入敏感信息
- 代付账号统一管理

## 禁止行为

| 禁止行为 | 原因 |
|----------|------|
| 输入客户个人信息（姓名、手机、身份证） | 隐私法规 |
| 输入客户商业数据 | 保密协议 |
| 输入密码、密钥、证书 | 安全风险 |
| 输入未公开的财务数据 | 合规要求 |
| 输入战略规划细节 | 商业机密 |

---

## 脱敏指南

如需使用 AI 处理敏感内容，先脱敏：

| 敏感信息 | 脱敏方式 |
|----------|----------|
| 姓名 | 替换为"张三"、"用户A" |
| 手机号 | 替换为 "138****1234" |
| 公司名 | 替换为"某公司"、"客户X" |
| 金额 | 使用相对值或示例值 |
| 日期 | 使用相对日期 |

## 合规要求

### 适用法规

| 法规 | 相关性 |
|------|--------|
| 《个人信息保护法》 | 个人信息处理 |
| 《数据安全法》 | 数据分类保护 |
| GDPR（如涉及） | 欧盟数据主体 |
| 行业规定 | 视具体行业 |

### 合规检查清单

- [ ] 数据已按级别分类
- [ ] AI 使用符合数据级别策略
- [ ] 敏感数据已脱敏
- [ ] 有审计日志可追溯

```
执行响应措施
    ↓
总结改进
```
